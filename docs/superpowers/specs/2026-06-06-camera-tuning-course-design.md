## 1. 目标与背景

- **当前角色**：手机测试工程师
- **目标岗位**：高通平台 Camera Tuning 工程师（3A 方向：AEC / AWB / AF）

## 2. 课程顶层设计

### 2.1 课程切分方案：「ISP Pipeline 顺流而下」（已选定）

按光线进入手机的真实顺序教学，建立直觉，避免一开始被术语劝退。

| 周次 | 天数 | 主题 | 核心知识点 |
|---|---|---|---|
| **Week 1** | Day 1-7 | **光学 + Camera 硬件入门** | 曝光三要素 AV/TV/SV、EV、像素 / Bayer 阵列 / CFA、CMOS 工作原理、镜头光圈 / 快门 / 马达、闪光灯类型、OTP |
| **Week 2** | Day 8-14 | **ISP Pipeline 全链路** | RAW→YUV→JPEG 全流程、高通 Spectra 架构、Chromatix 工具初识、各 ISP 模块作用（OB / Shading / Demosaic / CCM / Gamma / TMC / LTM / GTM 等） |
| **Week 3** | Day 15-21 | **AEC 第一阶段：基础** | 曝光表、Lux Index、亮度目标 (Target)、Safe Target 推导、收敛、抗带状、Bayer 网格 64×48 统计 |
| **Week 4** | Day 22-28 | **AEC 第二阶段：进阶** | 10 种 Scene Analyzers、ADRC / Dark Boost、3-EXP HDR、MFHDR / SHDR、Tone Mapping、双摄 AEC |
| **Week 5** | Day 29-35 | **AWB 第一阶段：基础** | AWB 流程、Bayer 网格统计、传感器 OTP 校准、参考点、光源权重向量、距离权重向量、误导色区 (MLC) |
| **Week 6** | Day 36-42 | **AWB 第二阶段：进阶** | 高级灰度世界 (AGW SA)、人脸辅助 (FA SA)、AWB 闪光灯、决策聚合器、三角增益 (TGA)、双摄同步、Fixed Color Matching、Smart Convergence |
| **Week 7** | Day 43-49 | **AF + Flash + HDR 综合** | 反差 AF (CAF)、Saliency AF、Depth AF、马达驱动、AF 预设 XML、闪光灯算法 (Precision Flash)、Auto Flicker Detection、滚动 / 静止带检测 |
| **Week 8** | Day 50-60 | **色彩科学 + 工具实操 + 面试冲刺** | LTM / GTM / TMC 关系、Gamma 优先级、多摄一致性、自研 Log / HDR 色彩管线、Chromatix 工具完整流程、模拟面试、简历润色 |

> Week 8 比其他周多 4 天（Day 50-60），用作冲刺与缓冲。

### 2.2 知识图谱（模块级，对应薄弱项粒度）

7 个一级模块，每个模块下挂多个子主题。**薄弱项分析以模块级粒度统计正确率**：

```
M1. 光学与硬件基础（Week 1）
M2. ISP Pipeline（Week 2）
M3. AEC 自动曝光（Week 3-4）
M4. AWB 自动白平衡（Week 5-6）
M5. AF 自动对焦（Week 7 上半）
M6. Flash + HDR + 色彩（Week 7 下半 + Week 8 上半）
M7. 工具与工程方法（Week 8 下半）
```

每个模块下的子主题、覆盖的 PDF 章节、对应面试题清单 → 在「附录 A：知识图谱明细」中详细列出（实施阶段补充）。

## 3. 系统架构

### 3.1 技术栈

- **形态**：纯静态 HTML/CSS/JS（无后端）
- **部署**：开发于 Mac，最终交付一个目录给用户复制到 Windows 学习
- **存储**：浏览器 `localStorage`（学习进度、笔记、考核记录、薄弱项统计）
- **路由**：客户端 Hash 路由（`#/today`、`#/notes/day-12` 等），单页应用
- **依赖**：无构建工具；HTML/CSS/JS 直接运行；可选 CDN 引入字体 / Markdown 渲染库

### 3.2 目录结构

```
3A-Course/
├── index.html                  # 单页应用入口（左导航 + 右内容）
├── assets/
│   ├── css/
│   │   ├── theme.css           # 深色专业风主题（基调色、字体）
│   │   └── components.css      # 卡片、按钮、表格、考核题样式
│   ├── js/
│   │   ├── app.js              # 路由 + 视图切换主控
│   │   ├── progress.js         # 解锁逻辑 + 学习进度持久化
│   │   ├── notes.js            # 笔记编辑 / 保存 / 导出
│   │   ├── quiz.js             # 考核出题 / 判分 / 自评
│   │   ├── report.js           # 日报生成
│   │   └── weakness.js         # 薄弱项统计 + 复习推荐
│   ├── img/                    # 图示、动图、Bayer 阵列示意等
│   └── data/
│       ├── curriculum.json     # 60 天课程大纲（结构化）
│       ├── days/
│       │   ├── day-01.json     # Day 1 内容（标题、章节、引用、配图、习题 ID）
│       │   ├── day-02.json
│       │   └── ...day-60.json
│       ├── quizzes/
│       │   ├── daily/          # 每日小测题库（按 day 分文件）
│       │   ├── weekly/         # 周考题库（按 week 分文件）
│       │   └── monthly/        # 月考题库（共 2 个）
│       └── interview/
│           └── questions.json  # 面试题库（含 PRD 已有题 + 我补充）
└── README.md                   # 用户使用说明（如何在 Win 上打开）
```

### 3.3 数据流

```
课程内容（curriculum.json + days/*.json）
    │
    ├─→ 总览页：渲染 60 天地图（已学/今天/锁定状态来自 localStorage）
    ├─→ 今日学习页：根据 progress.currentDay 加载对应 day-N.json
    │     └─→ 学完 → 触发 progress 推进 + 唤起日报弹窗
    ├─→ 笔记页：读写 localStorage.notes（按 day 分组）
    ├─→ 考核中心：从 quizzes/* 加载题目 → 提交后写入 localStorage.quizResults
    │     └─→ 累积统计 → weakness.js 计算模块级正确率
    ├─→ 日报页：聚合当天 progress + notes + quizResults 生成日报
    └─→ 薄弱项页：展示模块正确率 + 推荐复习章节链接
```

- **左侧导航**：固定 240px 宽，列出 6 大功能 + 8 周进度
- **右侧内容**：自适应宽度，通过 hash 路由切换
- **顶栏**：打卡连续天数、当前是 Day N
- **解锁逻辑**：未解锁的 Day 在导航中显示 🔒 灰色不可点

### 4.2 视觉风格（已选定：深色专业风）

- **主色**：背景 `#0f172a`（深蓝灰）/ 卡片 `#1e293b` / 高亮 `#3b82f6`（蓝）
- **文字**：主文 `#e2e8f0` / 次文 `#94a3b8` / 弱化 `#64748b`
- **字体**：中文主推 系统字 + Source Han Sans；标题用 600 字重；代码用 JetBrains Mono
- **图表**：用对比鲜明的高亮色（橙、绿、品红）便于参数曲线可读

> UI 细节交由 frontend-design skill 在实施阶段进一步落地（按钮、卡片、考核题样式等）。

### 4.3 6 个功能页面

#### 4.3.1 📚 总览页（`#/overview`）

- 顶部："你已完成 12/60 天，加油"
- 8 周地图（卡片网格）：每周 7 天小方块，已完成 = 蓝色实心 / 当前 = 蓝色边框 / 未解锁 = 灰色
- 点击已完成的天 → 进入回顾模式（只读）
- 点击当前天 → 跳到「今日学习」

#### 4.3.2 📅 今日学习页（`#/today`）

- 顶部：Day N · 主题 · 进度条
- 内容区：分章节展示（图文 + 动图 + 关键术语高亮）
- 章节内嵌「术语解释」浮窗（鼠标悬停在英文术语上）
- 底部：
  - 「今日小测」按钮（必须做完才能完成本日）
  - 「完成本日」按钮（点击后弹出日报抽屉 + 解锁下一天）
- **侧边浮动**：笔记快速记录浮窗（可折叠到右下角）

#### 4.3.3 📝 笔记页（`#/notes`）

- 列表视图：所有有笔记的 Day，按时间倒序
- 详情视图：Markdown 编辑器（支持基础语法）
- 操作：
  - 自动保存到 `localStorage`
  - 「📤 导出全部笔记」按钮 → 下载 `notes-YYYYMMDD.md`（合并所有 Day）
  - 「📤 导出 JSON」按钮 → 下载 `notes-YYYYMMDD.json`（含元数据，便于备份恢复）

#### 4.3.4 📊 考核中心页（`#/quiz`）

三个 Tab：
- **每日小测**：当前 Day 的小测（5-10 题，混合选择/填空/简答）
- **周考**：每周完成 Week N 后解锁（15-20 题）
- **月考**：第 30 天 / 第 60 天解锁（综合题，含面试题）

题型与判分：
- **选择题 / 填空题**：自动判分（含答案解释）
- **简答题**：用户作答 → 提交后展示参考答案 → 用户三选一自评：
  - 「我答对了」 / 「部分对」 / 「没答对」
  - 三档分别按 1.0 / 0.5 / 0 计入正确率

面试题专项：每周考核里穿插 2-3 道 PRD 已有面试题。

#### 4.3.5 📈 日报页（`#/report`）

两种触发：
- **完成本日时自动弹出**抽屉式日报
- **手动从导航打开**任意一天的历史日报

日报内容：
- 今日学习模块 + 用时
- 今日小测得分（按模块拆分）
- 今日笔记摘要（前 200 字）
- 模块累积正确率（与昨日对比）
- 一句鼓励语（基于进度模板生成）
- 「📤 导出今日日报」按钮（导出为 .md）

#### 4.3.6 🎯 薄弱项页（`#/weakness`）

- 7 个模块的正确率柱状图（按 M1-M7）
- 每个模块下方：
  - 正确率百分比
  - 错题最多的 3 个子主题
  - 「📖 推荐复习章节」 → 直接链接到对应 Day 的相关章节锚点
  - 「🔗 补充资料」 → 行业前辈笔记链接 / 网络公开资料链接 / PDF 章节页码

## 5. 内容生产策略

### 5.1 内容来源

| 来源 | 用途 |
|---|---|
| 7.0 Tuning Guide PDF（中文） | AEC / AWB / AF **操作步骤的主参考**（参数怎么调） |
| 3A 10 Deep Dive PDF（英文） | **架构 / 算法原理的主参考**（为什么这么调） |
| 行业前辈有道云笔记 | 实战经验、7450 平台特定细节 |
| 网络公开资料 | 光学基础、ISP 通用知识、Chromatix 教程视频 / 截图 |
| PRD 面试题 | 每周考核 + 月考嵌入 |

### 5.2 每日内容结构（对应 day-N.json schema）

```js
{
  id: "day-12",
  week: 3,
  module: "M3", // 用于薄弱项归类
  title: "AEC 曝光表调试",
  estimatedMinutes: 240,
  sections: [
    {
      id: "s1",
      title: "什么是曝光表",
      type: "concept",       // concept | procedure | example | recap
      content: "...",        // Markdown 文本
      images: ["exp-table-overview.png"],
      glossary: [
        { term: "Exposure Table", zh: "曝光表", explain: "..." }
      ]
    },
    ...
  ],
  references: [
    { source: "7.0 Tuning Guide", page: 14, section: "3.1 调试曝光表" },
    { source: "行业前辈笔记", url: "..." }
  ],
  dailyQuizId: "daily-day-12"
}
```

### 5.3 内容产出原则

- **从直觉到术语**：每个新概念先用生活类比（"曝光表就像菜谱"），再给出英文术语和中文释义
- **图示优先**：每个抽象概念至少配一张图 / 动图（手绘示意或 PDF 截图）
- **不教代码**：涉及代码片段时只解释"这段代码在做什么"，不要求用户写
- **面试题映射**：每个 Day 末尾标注"今天学的内容对应哪些面试题"

## 6. 考核与评估系统

### 6.1 三级考核

| 类型 | 频次 | 题量 | 题型分布 | 时长 |
|---|---|---|---|---|
| 每日小测 | 每天 | 5-10 题 | 选择 60% / 填空 20% / 简答 20% | 15-20 分钟 |
| 周考 | 每周末 | 15-20 题 | 选择 40% / 填空 20% / 简答 30% / 面试题 10% | 45-60 分钟 |
| 月考 | 第 30、60 天 | 30-40 题 | 综合 + 模拟面试题 | 90 分钟 |

### 6.2 判分与薄弱项联动

- 选择 / 填空：自动判分，写入 `quizResults`
- 简答：用户对照参考答案三档自评
- 每提交一次考核：
  1. 更新 `quizResults`
  2. 重算对应模块的累积正确率（写入 `weakness`）
  3. 若正确率低于 60%，在薄弱项页置顶提醒，并推荐复习章节

### 6.3 复习推荐规则

```
模块正确率 < 60%
    → 推荐：该模块所有 Day 的章节链接 + 行业前辈笔记片段 + PDF 章节页码

模块正确率 60-80%
    → 推荐：该模块错题最多的 3 个子主题对应的章节锚点

模块正确率 > 80%
    → 显示"已掌握"标记，不主动推荐
```

## 7. 错误处理与边界

| 场景 | 处理 |
|---|---|
| `localStorage` 满了（理论 5-10MB 限制） | 自动提示导出备份 + 清理 60 天前的 quizResults 详情（保留汇总） |
| 用户清浏览器缓存导致进度丢失 | 提供「📥 导入备份 JSON」按钮，从导出文件恢复 |
| 用户跨电脑（Mac → Win） | README 说明：先在 Mac 上导出 JSON 备份 → 复制课程目录到 Win → 在 Win 浏览器导入 |
| 当天没学完想关闭浏览器 | 进度自动保存；下次打开停在最近滚动位置（记录 `lastSection`） |
| 当天超前学习（周末多学） | 允许；只要每天点了「完成本日」就推进 progress |
| 跨日不打卡 | streak 重置为 0；不锁内容（用户进度不依赖打卡） |

## 9. 不在本设计范围内（YAGNI）

- 后端 / 多设备同步（用户明确只在一台电脑）
- 真实视频内容（PRD 提到"图文并茂、视频或动图"，本设计交付**图文 + 动图**；视频成本过高且自制困难）
- 编程练习模块（用户明确不学代码）
- 自适应推送 AI 推荐（薄弱项推荐用规则即可，不引入算法模型）
- 移动端响应式（用户在 Win 桌面学习，桌面优先）
- 多用户 / 账号系统

## 10. 风险与缓解

| 风险 | 缓解策略 |
|---|---|
| 内容量大（60 天 × 4 小时 ≈ 240 小时教学内容），生产周期长 | 优先输出骨架 + Week 1-2 完整内容验证体验，再批量生产 |
| 用户阅读体验如果太枯燥会失去动力 | 每日内容控制在 3-4 节小段；引入比喻、动图；每日打卡 + streak 激励 |
| 简答题用户自评不准 | 提供详细的"参考答案要点列表"，让用户对照打钩，减少主观偏差 |
| Mac 和 Win 浏览器渲染差异 | 用 Edge / Chrome 内核为主测试目标；不使用平台特有 API |
| `localStorage` 数据丢失 | 强制每周提示一次导出备份；日报页突出"最近一次备份" |
